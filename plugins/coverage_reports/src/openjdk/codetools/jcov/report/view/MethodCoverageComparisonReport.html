Method coverage comparison report

This report compares method coverage between two code coverage collections. A target coverage is compared to a reference coverage. Coverage of every method from the target collection is then compared to coverage of the same method in the reference collection.

A typical use for the report would be to investigate changes in method coverage over time, in which case the reference collection would be performed in the past and the target collection is recent.

Every method in the report marked in one of four ways:

  • Covered - methods that are covered in the target coverage collection
  • Uncovered - methods that are not covered in either target or reference collections
  • Lost - methods that are covered in the reference collection but are not covered in the target collection
  • New - methods that are present in the target collection and missing in the reference collection

In addition to the method coverage, the report provides line coverage from the target collection.

Folder view

A folder view lists total number of method for all four groups of methods for the files nested hierarchically within this folders.

Next the view shows direct sub-folders of this folder along with the totals for the groups of methods and total line coverage. The table is sortable by any column. Clicking on a sub-folder name takes to a folder view for that sub-folder. The table may be omitted altogether should tere be no direct sub-folders.

Next the view shows direct files which are located directly in this folder along with the totals for the groups of methods and total line coverage for the file. The table is sortable by any column. Clicking on a file name takes to a file view for that file.

File view

A file view lists methods of the file, colored in accordance to the groups to which the methods belong. Clicking on a method signature takes to the first line of the method, according to the line table from the bytecode.

Next there is the file source code. The source code is colored according to line coverage from the target collection:

  • covered
  • uncovered
  • no information on the text

Every line of the source code, which, according to the line table, is nested within one or more method, is prepended by one or more links, which lead to a corresponding method at the top of the file view. Every such link is colored the same way as the method it links to. Hovering mouse over such link displays method signature to which the link corresponds.
